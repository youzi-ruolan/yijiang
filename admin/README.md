# Colorist Admin

这是一个和当前小程序仓库同目录的轻量级后台管理原型，特点：

- 零额外依赖
- 直接复用 `pages/home/mock.js` 的首页运营数据结构
- 使用浏览器 `localStorage` 保存后台改动
- 支持商品管理、分类管理、内容运营、订单概览、系统设置

## 为什么这样实现

当前仓库是 **微信小程序模板**，没有现成的 Web 后台工程；同时为了快速落地、方便你继续迭代，我先实现了一个：

- **可直接运行**
- **和现有 mock 数据结构一致**
- **无需安装新依赖**

的后台管理系统骨架。

## 推荐的正式后台方向

如果后面要接真实接口、角色权限、菜单路由、表单引擎、图表和部署，我推荐升级到：

- `TDesign Vue Next Starter`

原因：

1. 和当前项目都属于 TDesign 设计体系，视觉统一成本最低
2. 更适合做商品、订单、内容、用户等标准后台
3. 后续接接口、登录鉴权、菜单权限会更顺手

## 本地运行

在仓库根目录执行：

```bash
node admin/server.js
```

然后打开：

```text
http://127.0.0.1:4173/admin/
```

## 当前后台包含的模块

- 项目总览
- 商品管理
- 分类管理
- 内容运营
- 订单管理
- 系统设置

## 数据说明

- 默认数据来源：`pages/home/mock.js`
- 本地编辑后的数据保存在浏览器 `localStorage`
- 可以通过页面上的“复制当前数据 / 导出 JSON”能力做版本管理或后续接接口
